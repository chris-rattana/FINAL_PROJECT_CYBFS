# Presentation Assets README

## Purpose
This folder stores the visual and supporting assets used for the final oral presentation.

Its purpose is to centralize reusable materials that make the final presentation clearer, cleaner, and easier to maintain.

These assets support the deck and oral explanation without overloading the main technical folders.

## Expected Content
This folder may contain:
- final architecture visuals
- simplified diagrams for slides
- screenshots reused in the presentation
- icons or cleaned visuals
- summary images
- presentation-ready exports from other evidence folders

## Recommended File Types
Typical file formats may include:
- `.png`
- `.jpg`
- `.svg`
- `.pdf`

## Recommended Naming Convention
Use clear filenames such as:
- `PRES01_architecture_visual.png`
- `PRES02_security_controls_visual.png`
- `PRES03_blue_team_summary.png`
- `PRES04_red_team_summary.png`
- `PRES05_monitoring_visual.png`

## Quality Rules
Each asset should:
- be visually clean
- be readable on a slide
- support one clear presentation message
- avoid unnecessary technical clutter
- remain consistent with the actual project content

## Notes
This folder is especially useful for keeping slide visuals separate from raw technical proof while still staying linked to the real project evidence.
