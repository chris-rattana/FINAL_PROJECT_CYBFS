# Deployment Screenshots README

## Purpose
This folder stores screenshots proving that infrastructure components were deployed and are usable.

Its purpose is to centralize visual proof of:
- running services
- application pages
- administrative interfaces
- platform dashboards
- successful service access

These screenshots support deployment validation and final review.

## Expected Content
This folder may contain:
- web application screenshots
- Nextcloud interface screenshots
- Wazuh interface screenshots
- admin dashboard screenshots
- setup confirmation pages
- screenshots of successful access to deployed services

## Recommended File Types
Typical file formats may include:
- `.png`
- `.jpg`
- `.pdf`

## Recommended Naming Convention
Use clear filenames such as:
- `DEP01_web_homepage.png`
- `DEP02_nextcloud_login.png`
- `DEP03_nextcloud_dashboard.png`
- `DEP04_wazuh_dashboard.png`
- `DEP05_admin_interface.png`

## Quality Rules
Each screenshot should:
- show the useful part clearly
- avoid unnecessary desktop clutter
- be readable without excessive zoom
- not expose sensitive secrets
- correspond to a real deployment validation point

## Notes
Screenshots stored here can later be reused in deployment proof, Blue Team evidence, Red Team context, and final presentation slides.
