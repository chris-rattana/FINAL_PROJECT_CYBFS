# Architecture Diagrams README

## Purpose
This folder stores the architecture diagrams used in the final CYBFS project.

Its purpose is to preserve the visual materials that explain:
- the global infrastructure structure
- the logical relationship between services
- trust boundaries
- network exposure logic
- service-to-service flows

These diagrams support the technical specification, deployment documentation, and final presentation.

## Expected Content
This folder may contain:
- global architecture diagram
- logical architecture diagram
- trust-zone diagram
- network flow diagram
- service interaction diagram
- simplified presentation-ready diagram versions

## Recommended File Types
Typical file formats may include:
- `.png`
- `.jpg`
- `.pdf`
- `.drawio`
- `.svg`

## Recommended Naming Convention
Use clear filenames such as:
- `ARCH01_global_architecture.png`
- `ARCH02_logical_architecture.png`
- `ARCH03_trust_zones.png`
- `ARCH04_service_flows.png`
- `ARCH05_presentation_version.png`

## Quality Rules
Each diagram should:
- be readable
- match the documented architecture
- use consistent naming
- clearly distinguish exposed, internal, and admin layers
- remain reusable for both documentation and slides

## Notes
Prefer one clean source file when possible, plus one exported image version for reuse in presentation materials.
