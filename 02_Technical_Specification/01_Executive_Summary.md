# Executive Summary

## Document Purpose
This document presents the technical specification of the infrastructure designed for the final CYBFS project.

The project consists of designing, documenting, securing, and assessing a realistic SME infrastructure for a remote-first startup. The objective is to provide a coherent and defensible environment that supports remote work, centralized administration, secure access to business resources, operational continuity, and basic cyber resilience.

## Project Context
The selected company profile is a remote-first startup. This type of organization depends on digital services to enable employees to work from different locations while maintaining access to internal resources, shared files, business applications, and administrative systems.

This context creates several technical and security needs:
- secure remote access
- centralized identity and access management
- controlled exposure of services
- file sharing for distributed teams
- reliable application and database hosting
- backup and monitoring capabilities
- practical baseline security controls

## Project Objective
The objective of the project is to design and document an infrastructure that is:
- functional
- secure
- integrated
- manageable
- adapted to SME constraints

The infrastructure must support the minimum required services of the final project while remaining realistic in scale and complexity.

## Infrastructure Overview
The target infrastructure includes the following core components:
- a web server or web application
- a database securely connected to the web service
- a file-sharing service
- a centralized identity and access management component
- a backup and monitoring capability
- baseline security controls including access restriction, firewalling, and service hardening

The infrastructure is designed as a small-scale but professional environment that can be explained, demonstrated, assessed, and improved.

## Security Orientation
Security is treated as an integrated part of the infrastructure rather than an afterthought. The project focuses on reducing unnecessary exposure, improving visibility, controlling access, and documenting weaknesses and remediations through both Blue Team and Red Team deliverables.

The main security principles applied in this project are:
- least privilege
- controlled access
- service exposure reduction
- basic segmentation
- logging and monitoring
- recoverability through backup logic
- traceability through documentation and evidence

## Expected Benefits
The proposed infrastructure is intended to provide the following benefits:
- improved control over user access
- clearer administration of identities and permissions
- more reliable service delivery
- better data protection and recovery readiness
- stronger visibility over operational and security events
- a structured basis for security assessment and remediation

## Deliverable Positioning
This technical specification serves as the architectural and documentary backbone of the project. It explains the design choices, security logic, risk posture, and operational assumptions behind the deployed environment.

It is complemented by:
- infrastructure deployment documentation
- project management artifacts
- Blue Team documentation
- Red Team documentation
- technology watch report
- final presentation materials

## Conclusion
This technical specification defines a realistic and secure infrastructure for a remote-first startup. The project is intentionally scoped to remain feasible, demonstrable, and coherent, while still covering the essential operational and cybersecurity requirements expected in the final CYBFS project.
