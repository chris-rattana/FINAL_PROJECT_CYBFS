# Company Profile

## Selected Company Type
Remote-first Startup

## Business Description
The company is a small-to-medium startup operating primarily with remote employees. Team members work from different locations and rely on centralized digital services to collaborate, access internal resources, share files, manage business applications, and maintain operational continuity.

The company needs a secure, reliable, and scalable infrastructure that supports remote work while reducing exposure to common cyber risks such as unauthorized access, weak identity controls, data leakage, service disruption, and insufficient visibility over infrastructure events.

## Why This Profile Was Selected
This profile is highly relevant for a cybersecurity and infrastructure project because it naturally requires:
- secure remote access
- centralized identity and access management
- protected web and database services
- file sharing for distributed teams
- backup and monitoring
- practical security controls adapted to SME constraints

It is also a realistic scenario for a modern organization with limited resources but strong operational and security needs.

## Organizational Context
The startup has a limited technical team and must deploy a cost-conscious but professional infrastructure. The environment must remain manageable, documented, and defensible during security review.

The company cannot rely on informal access or unmanaged devices alone. It needs a structured infrastructure with:
- controlled user access
- central administration
- secured internal resources
- monitored services
- documented recovery capabilities

## Main Business Needs
- allow remote users to securely access company resources
- centralize user and access management
- host a web service connected to a database
- provide secure file sharing and collaboration
- maintain service availability and data protection
- detect issues and support response through logs and monitoring

## Security Priorities
- strong access control
- remote access protection
- service exposure reduction
- backup and recovery readiness
- event visibility and monitoring
- basic segmentation and hardening

## Constraints
- SME-scale budget
- limited project duration
- realistic implementation scope
- need for clear documentation
- balance between security, usability, and maintainability

## Expected Outcome
The final outcome is a documented, functional, and defensible infrastructure proposal and deployment for a remote-first startup, supported by technical evidence, project management material, security assessment, and a final presentation.
